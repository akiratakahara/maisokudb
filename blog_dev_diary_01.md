# マイソクDB開発日記 #1 — 「マイソクを撮るだけ」で不動産投資判断ができるアプリを作っている話

## なぜ作り始めたか

不動産投資の検討って、とにかく情報収集が大変です。

マイソク（物件概要書）を受け取ったら、まず手入力でExcelに転記。次に国土交通省のサイトで周辺の取引事例を調べ、ハザードマップを確認し、複数の銀行に融資条件を問い合わせ、利回りを計算し...

**この作業、全部自動化できるのでは？**

そう思って開発を始めたのが「マイソクDB」です。

---

## 今日までに実装した機能

### AI自動抽出
マイソクのPDFやスクリーンショットをアップロードするだけで、Google Gemini AIが物件情報を自動で読み取ります。価格、利回り、面積、構造、最寄駅...手入力ゼロ。

### 11種類の公的データ自動取得
物件の住所から、以下のデータを自動で引っ張ってきます：

| データ | ソース |
|---|---|
| 周辺取引事例 | 国土交通省 不動産情報ライブラリ |
| 地価公示 | 同上 |
| 洪水浸水想定 | 同上 |
| 土砂災害警戒区域 | 同上 |
| DID（人口集中地区） | 同上 |
| 将来推計人口（2050年） | 同上 |
| 駅乗降客数 | 同上 |
| 標高 | 国土地理院 |
| 生活利便施設 | OpenStreetMap |
| 空き家率 | e-Stat（総務省統計） |
| 人口移動（転入転出） | 同上 |

### 16行の銀行融資シミュレーション
物件情報を入力すると、登録済み16行の銀行から融資可能な候補を自動選定。月額キャッシュフロー、減価償却、節税効果まで一括比較。

### AI投資分析
ボタン1つで、AIが公的データを全部読み込んだ上で物件を4軸（立地・収益性・将来性・流動性）で5段階評価。空き家率や人口動態も加味した分析コメントを生成。

### コミュニティ比較データ
ユーザーが物件を登録するほどデータが蓄積。「この駅周辺の平均価格は○万円」「相場より○%安い」がわかる。

### PC管理画面
レインズから取得した複数物件PDFを一括でAI抽出・DB登録。CSVエクスポートも対応。

## 苦労したポイント

### 住所→座標の精度問題
マイソクの住所表記は省略されていることが多い。最初はNominatim（OpenStreetMap）を使っていたが、日本の住所に弱くてマップのピンがズレる。**国土地理院の地理院地図API**に切り替えたら精度が大幅改善。

住所の座標がズレると、DID判定も洪水リスクも地価公示も全部ズレるので、ここは根幹。

### レインズPDFの一括取り込み
レインズから複数物件をまとめてダウンロードすると、1つのPDFに複数のマイソクが入っている。ページの区切りをAIに判定させて、物件ごとに分離・抽出する仕組みを作った。

### App Store審査
TestFlightビルドの署名問題（"No Team Found in Archive"）で半日ハマった。`CODE_SIGNING_ALLOWED=NO`でビルドしてPlistBuddyでTeamIDを注入する方法で解決。
